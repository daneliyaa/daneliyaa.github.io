hexo+GitHub博客搭建

搭建过程
第一步：首先安装Node.js和npm：由于Hexo是基于Node.js的静态博客框架，因此需要先安装Node.js和npm。
另外还需安装Git：Hexo使用Git来进行版本控制和部署。
以下是安装前置软件包
Git https://git-scm.com/download/win
VSCode https://code.visualstudio.com/
Node.JS https://nodejs.org/en
Node.js安装完成后，通过以下命令可用来确认安装成功：、
node -v
npm –v
安装Git安装完成后，通过以下命令可用来确认安装成功：
git –version
第二步：安装hexo

  1. 更改npm镜像源
    进入cmd
    npm config set registry https://registry.npmmirror.com
  2. 创建一个文件夹 在里面打开终端
    第三步：使用npm安装hexo
    npm install -g hexo-cli
    第四步：创建博客目录
    hexo init my-blog
    cd my-blog
    npm install
    第五步：启动本地服务器：
    hexo server
    打开浏览器，访问 http://localhost:4000，即可看到默认的Hexo博客页面。
    第六步：配置Hexo
    1.修改配置文件：编辑_config.yml文件，设置站点信息。
    title: illumi激推’ blog

subtitle: illumi激推’ blog
description:
author: daneliya
language: en
timezone: ‘’

搭建过程所遇问题以及解决方法
1．安装Node.js或者npm失败
解决方案：确保是从Node.js官网下载安装包，并按照系统要求正确安装；可以尝试使用包管理器安装，如Homebrew（macOS）或者Chocolatey（Windows）。
2.hexo-cli安装失败
解决方案：尝试使用管理员权限安装：
sudo npm install -g hexo-cli
检查npm配置，确保网络通畅，或使用淘宝镜像：
npm config set registry https://registry.npm.taobao.org
3.hexo generate`报错
解决方案：删除缓存和已生成的文件，然后重新生成：
hexo clean
hexo generate
检查文章文件的格式和命名是否正确，确保Markdown文件中没有语法错误。

虚拟机搭建

一．Ubuntu镜像下载
打开官网链接，下载Ubuntu镜像（Ubuntu Server）
以下是网址：https://ubuntu.com/download/server
二．创建虚拟机
1.选择创建新的虚拟机，进入新建虚拟机向导；
2选择自定义类型配置，选择“下一步“；
3选择硬件兼容性（大家根据自己的VM版本选择对应的就好），选择“下一步“;
4选择稍后安装操作系统，然后“选择下一步“；
5选择客户机操作系统：(我这里选择的“客户机操作系统“是Linux，”版本”是Ubuntu 64位),接着选择”下一步“；
6命名虚拟机：先命名虚拟机名称，可创建一个目录，指定此为位置路径，后将虚拟机名称ctrl+c再ctrl+v于目录后。接着选择“下一步“;
7 指定磁盘容量：我这里选择的是针对Ubuntu 64位的建议大小：20GB，选择“将虚拟磁盘拆分成多个文件“，然后”下一步“;
8 自定义硬件：选择“自定义硬件“，接着选择”启动时连接“，选择”新CD/DVD(SATA)”,选择“使用ISO映像文件”，打开“浏览”，找到下载好的Ubuntu镜像文件，选择打开，回到“硬件”页面“确认无误后选择”关闭“，选择”完成“;
三．安装系统
1 选择打开虚拟机
2 等待界面加载完毕，点击鼠标进入界面，选择“Try or Install Ubuntu Server”,按回车键；
3 选择“English”，按回车键；
4 进入“Identify keyboard”界面选择“done”，继续按回车键；
5 进入“Choose the type of install”界面，”choose the base for the installation“选择“Ubuntu Server”,”Additional options”不用选，选择“done”，按回车键；
6 进入“Network configuration”界面，选择”done”按回车键；
7 进入“Proxy configuration”界面，先不用输入”Proxy address”,选择“done”,继续回车键；
8 进入“Ubuntu archive mirror configuration”界面，镜像地址测试，当界面显示“This mirror location passed tests.”以及方框内显示出“Reading package lists…”，即通过测试，接着选择”done“，按回车键；
9 进入“Guided storage configuration”界面，于“Configure a guided storage layout,or creat a custom one:”选择“Use an entire disk”以及”Set up this disk as an LVN group”,然后选择“done”，继续回车键；
10 进入“Storage configuration”界面，选择”done“按回车键；
11 弹出“Confirm destructive action”界面，选择”continue“，按回车键；
12 进入“Profile configuration” 界面，输入对应信息“name” “server name” “pick a username” “password” “confirm your password”,输入完后，选择”done“按回车键；
13 进入“Upgrade to Ubuntu Pro” 界面，可选择“skip for now”(我选择这个)，也可以选择”Enable Ubuntu Pro”，然后选择”continue “按回车键；
14 进入“SSH configuration” 界面，可选择”Install OpenSSH server”也可以不选(我选择不选)，选择”done“按回车键；
15 进入“Featured server snaps” 界面，选择”done“按回车键；
16 进入“Installation complete” 界面，等待弹出“subiquity/Late/run:”，选择“Reboot Now”,按回车键；
17 等待弹出“Please remove the installation medium,then press ENTER:“按回车键，重启虚拟机安装Ubuntu；
18 等待加载完毕，结尾弹出“Used fallback datasource”, 按回车键；
19 输入用户名称，按回车键，然后再输入密码，按回车键；
20 结尾显示“a@a:~$”即安装完毕！
